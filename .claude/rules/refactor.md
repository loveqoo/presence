# 리팩토링 규칙

## 복잡도 기반 리팩토링

리팩토링 전 복잡도를 측정하고 데이터 기반으로 우선순위를 정한다. 복잡도가 높은 파일부터 착수한다.

| 유형 | 전략 |
|------|------|
| 긴 조건 체인 | 맵/테이블 디스패치, 다형성 |
| 깊은 중첩 | 조기 반환, 함수 추출 |
| 복합 조건 | 술어 함수 추출 |
| 긴 함수 | 단일 책임 단위로 분리 |
| 콜백 중첩 | 파이프라인 합성, 모나드 체이닝 |
| 같은 조건 반복 | 한 곳으로 응집 (전략 패턴, ADT 매칭) |

## 구조 우선 원칙

구현의 효율성보다 구조가 주는 효율성을 우선한다. 지금 비어 있는 껍데기라도, 전체를 일관되게 형성하는 구조라면 유지한다. 같은 레벨의 관심사가 서로 다른 형태를 가지면 소비처에 분기가 생긴다.

## 타입 분기 → 다형성

타입 플래그로 분기하는 코드가 보이면 서브클래스로 전환한다. 공통 클래스에서 알고리즘을 정의하고, 유형별 서브클래스에서 각 단계를 구현한다.

## Extract Class 판단 기준

다음 신호가 보이면 Extract Class를 적용한다.

| 신호 (Code Smell) | 판단 기준 | 조치 |
|---|---|---|
| **Long Parameter List** | 함수 파라미터 5개 초과 | 공유 설정을 인스턴스 필드로 |
| **Data Clump** | 같은 묶음이 반복 (파라미터, 맵 키셋, 설정 그룹 등 형태 무관) | 묶음을 클래스로 응집 |
| **Pass-through** | 함수 A가 받은 인자를 거의 그대로 함수 B에 전달 | A와 B를 같은 클래스의 메서드로 |
| **Feature Envy** | 여러 함수가 같은 상태의 필드를 반복 참조 (독립 함수, 맵 값, 콜백 등 형태 무관) | 그 상태를 가진 클래스의 메서드로 이동 |

## Extract Class 적용 원칙

- 클래스는 설정 응집 + 메서드 그룹화 용도. 상속 계층은 최소화
- 순수 로직은 메서드 안에서도 FP 규칙 준수 — Either, Free, Writer 등 모나드 사용 유지
- constructor에서 설정, 메서드에서 행위 — 설정 pass-through 제거
- 이중 패턴(`x`/`getX`)은 통일 — `resolveX` 하나로 수렴

## 모듈 스코프 최소화

모듈 스코프의 독립 함수는 최소화한다. 클래스 인스턴스를 통해서만 사용되는 함수는 해당 클래스의 메서드로 흡수한다.

## 클래스 분리 판단 기준

한 클래스가 독립적으로 변경될 수 있는 여러 책임을 가지면, 책임별로 별도 클래스로 분리하고 조율 객체가 조합한다.

## 네이밍

- 언더바 접두사(`_name`) 금지 — 함수/메서드 내부의 일시적 변수는 예외
- 함수/메서드 이름은 구현 내용을 충분히 반영해야 한다
- 파라미터 순서: 반복 사용되거나 필수인 파라미터를 앞쪽에 배치
- 파라미터가 많으면 필수와 옵션을 구분하여 격리한다 (필수는 positional, 선택은 옵션 객체)
- 대문자: 불변 값 (열거 태그, 상수), 파스칼케이스: 생성자 (클래스, 팩토리), 카멜케이스: 함수·변수·인스턴스
- 개발자만 보는 에러 메시지는 영어로 작성한다
- 한 글자 변수명(`s`, `e`, `x` 등) 금지 — 의미를 드러내는 이름을 사용한다

## 모듈 분리 판단 기준

- 한 파일에 관심사가 다른 함수/상수가 나열되면 모듈 분리
- 정책 상수는 `policies.js`에 통합. 파일별 로컬 상수 금지
- 유틸리티 함수가 `fun-fp.js`에 이미 존재하면 재정의하지 않고 import
- re-export로 창구 역할을 하는 파일 금지 — 소비처가 원천 모듈에서 직접 import
- import한 객체를 re-export 금지 — 모듈은 자신이 정의한 것만 export
- 사용하지 않는 import 객체 제거
- 사용하지 않는 함수/메서드는 제거하거나 사용하는 곳으로 옮긴다
- 테스트에서만 사용되는 함수/메서드는, 다른 public API로 동일한 검증이 가능하면 제거한다
- import 구문 사이에 코드를 넣지 않는다
- 필요 없는 기능은 제거한다
- 한 곳에서만 사용되는 함수는 사용처에 인라인한다
- 상태 분기(switch case, 메시지 핸들러 등)에는 "이 상태에서 무엇을 하는가"를 한 줄 주석으로 명시한다
