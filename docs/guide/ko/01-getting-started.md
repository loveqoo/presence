# 시작하기

이 문서는 presence를 처음 사용하는 분을 위한 안내입니다.  
설치부터 첫 화면이 뜨는 순간까지, 단계별로 따라 하실 수 있습니다.

---

## presence가 무엇인가요?

presence는 **개인 업무 대리 에이전트 플랫폼**입니다.  
말로 지시하면 AI가 대신 일해 줍니다. 파일을 읽고 쓰거나, 웹 페이지를 찾아보거나, 셸 명령을 실행하거나, 외부 도구와 연결하는 일들을 대화 형식으로 처리할 수 있습니다.

화면은 터미널(명령 프롬프트 창)에서 동작하며, 채팅 앱처럼 입력창에 글을 쓰고 Enter 를 누르는 방식으로 사용합니다.

presence는 서버와 클라이언트(화면)로 나뉩니다.

- **서버** — AI와 실제 작업을 처리하는 두뇌. 백그라운드에서 실행됩니다.
- **클라이언트(TUI)** — 화면에 보이는 채팅 창. 서버에 연결해서 사용합니다.

같은 컴퓨터에서 서버와 클라이언트를 모두 실행하는 것이 기본 방식입니다.

---

## 준비물

- Node.js가 설치된 컴퓨터
- presence 소스 코드 (압축 해제 또는 git clone 완료)
- LLM API 키 (예: OpenAI `sk-...`)

터미널(macOS는 터미널 앱, Windows는 PowerShell 또는 Git Bash)을 열고, presence 폴더로 이동한 뒤 아래 단계를 따라 주세요.

---

## 1단계 — 의존성 설치

```bash
npm install
```

처음 한 번만 실행하면 됩니다. 완료되면 다음으로 넘어갑니다.

---

## 2단계 — 설정 파일 만들기

```bash
cp config.example.json ~/.presence/config.json
```

복사된 파일(`~/.presence/config.json`)을 텍스트 편집기로 열고, LLM API 키를 입력합니다.

**최소 설정 예시:**

```json
{
  "llm": { "apiKey": "sk-여기에-API-키를-입력하세요" }
}
```

파일을 저장하면 설정이 완료됩니다.

---

## 3단계 — 사용자 등록

presence는 계정 없이는 사용할 수 없습니다. 처음에는 첫 번째 사용자를 등록해야 합니다.

```bash
npm run user -- init --username 내이름
```

`내이름` 부분에 사용하실 이름을 넣으세요. 영문자, 숫자, `_`, `-` 만 사용 가능하며 64자 이내여야 합니다.  
예: `npm run user -- init --username alice`

> 나중에 사용자를 추가하려면 `npm run user -- add --username 다른이름` 을 사용합니다.  
> 등록된 사용자 목록을 보려면 `npm run user -- list` 를 사용합니다.

이 명령은 비밀번호를 임시로 설정하고, 첫 로그인 시 변경하도록 안내합니다.

---

## 4단계 — 서버 시작

```bash
npm start
```

아래와 같이 출력되면 서버가 정상적으로 실행된 것입니다.

```
Presence server listening on http://127.0.0.1:3000
```

서버는 이 터미널 창에서 계속 실행 중이어야 합니다. **이 창을 닫으면 서버도 종료됩니다.**  
다른 터미널 탭 또는 새 창을 열고 5단계로 넘어가세요.

---

## 5단계 — TUI 클라이언트 실행

서버가 실행 중인 상태에서, 새 터미널 창에서 presence 폴더로 이동한 뒤 실행합니다.

```bash
npm run start:cli
```

클라이언트가 시작되면 터미널에 아래와 같은 프롬프트가 나타납니다.

```
사용자명:
```

---

## 6단계 — 로그인

### 사용자명과 비밀번호 입력

프롬프트에 3단계에서 등록한 사용자명을 입력하고 Enter 를 누릅니다.

```
사용자명: alice
비밀번호:
```

비밀번호 입력 시 화면에 글자가 보이지 않을 수 있습니다 (보안상 가려집니다). 그대로 입력하고 Enter 를 누르세요.

로그인에 실패하면 "다시 시도하세요." 메시지와 함께 재입력을 요청합니다. **최대 3번** 시도할 수 있으며, 3번 모두 실패하면 프로그램이 종료됩니다.

### 최초 로그인 — 비밀번호 변경

처음 등록한 계정은 **반드시 비밀번호를 변경해야** 합니다. 로그인에 성공하면 자동으로 아래 화면이 나타납니다.

```
[alice] 최초 로그인입니다. 새 비밀번호를 설정하세요.
새 비밀번호:
새 비밀번호 확인:
```

새 비밀번호를 두 번 입력하세요. 조건:

- 8자 이상
- 두 번 입력한 값이 같아야 함

비밀번호 변경에 성공하면 `"비밀번호가 변경되었습니다."` 라고 표시되고 바로 채팅 화면으로 진입합니다.

> 비밀번호 변경도 최대 3번 시도할 수 있습니다. 3번 모두 실패하면 프로그램이 종료되며, 다시 `npm run start:cli` 를 실행해서 재시도하면 됩니다.

---

## 7단계 — 첫 화면 확인

로그인에 성공하면 채팅 화면이 나타납니다. 화면은 아래와 같이 구성되어 있습니다.

```
┌─────────────────────────────────────────────────────────────────┐
│  (대화 내용이 이 영역에 표시됩니다)                             │  ← ChatArea
│                                                                 │
│                                                                 │
├─────────────────────────────────────────────────────────────────┤
│  > 여기에 입력하세요                                            │  ← InputBar
├─────────────────────────────────────────────────────────────────┤
│  idle │ session: alice-default │ budget: 0% │ gpt-4o │ main   │  ← StatusBar
└─────────────────────────────────────────────────────────────────┘
```

| 영역 | 역할 |
|------|------|
| **ChatArea** (위쪽 넓은 영역) | AI의 답변과 대화 이력이 쌓입니다 |
| **InputBar** (아래 입력창) | 메시지를 입력하는 곳입니다. Enter 로 전송합니다 |
| **StatusBar** (맨 아래 한 줄) | 현재 상태, 사용 중인 AI 모델, 폴더, 브랜치 등 정보를 보여 줍니다 |

**SidePanel** (오른쪽 패널)은 기본적으로 숨겨져 있으며, `/statusline panel` 슬래시 커맨드로 열 수 있습니다. 사용 중인 도구 목록, 메모리 수, 이벤트 등을 확인할 수 있습니다.

### 유용한 단축키

| 키 | 동작 |
|----|------|
| `Enter` | 메시지 전송 |
| `Esc` (AI가 작업 중일 때) | 작업 취소 |
| `Esc` (평상시) | 임시 알림 메시지 닫기 |
| `Ctrl + C` | 프로그램 종료 |
| `Ctrl + T` | 내부 작업 기록(트랜스크립트) 열기/닫기 |
| `Ctrl + O` | 도구 상세 정보 펼치기/접기 |
| `↑ / ↓` | 이전에 입력했던 내용 불러오기 |

이제 입력창에 원하는 내용을 입력하면 됩니다.  
슬래시(`/`) 를 입력하면 사용 가능한 커맨드 목록이 나타납니다. 자세한 내용은 [04-slash-commands.md](04-slash-commands.md)를 참고하세요.

## 서버 URL 바꾸기 (선택)

기본적으로 클라이언트는 같은 컴퓨터의 서버(`http://127.0.0.1:3000`)에 연결합니다.  
서버가 다른 주소에서 실행 중이라면 아래 두 가지 방법으로 지정할 수 있습니다.

**방법 1 — 실행할 때 직접 지정:**

```bash
npm run start:cli -- --server http://192.168.1.10:3000
```

또는 등호(`=`) 형식:

```bash
npm run start:cli -- --server=http://192.168.1.10:3000
```

**방법 2 — 환경 변수로 지정:**

```bash
PRESENCE_SERVER=http://192.168.1.10:3000 npm run start:cli
```

환경 변수를 항상 유지하고 싶다면 셸 설정 파일(`.bashrc`, `.zshrc` 등)에 `export PRESENCE_SERVER=...` 를 추가하세요.

> 두 방법을 동시에 사용하면 `--server` 인수가 우선합니다.

## 연결이 안 될 때 확인 사항

### "서버에 연결할 수 없습니다" 메시지가 나온다

클라이언트를 실행했을 때 아래와 같은 메시지가 나오면:

```
서버에 연결할 수 없습니다: http://127.0.0.1:3000
서버가 실행 중인지 확인하세요: npm start
```

아래 사항을 순서대로 확인해 보세요.

**체크리스트:**

- [ ] 서버를 실행한 터미널 창이 아직 열려 있나요?
      서버 창을 닫으면 서버도 종료됩니다. `npm start` 를 다시 실행하세요.
- [ ] 서버가 실행된 주소와 클라이언트가 연결하려는 주소가 같나요?
      서버 창에 표시된 주소(`http://127.0.0.1:3000`)와 클라이언트의 `--server` 값을 비교해 보세요.
- [ ] 방화벽이나 네트워크 설정이 해당 포트를 막고 있지 않나요?
      원격 서버에 연결하는 경우에는 서버 컴퓨터의 방화벽 설정을 확인하세요.

### 로그인 화면이 나오지 않고 바로 종료된다

서버에 등록된 사용자가 없는 경우 서버 자체가 시작을 거부합니다.  
3단계(사용자 등록)를 먼저 완료했는지 확인하세요.

### 로그인 3번 실패 후 종료된다

`npm run start:cli` 를 다시 실행하면 재시도할 수 있습니다.  
비밀번호가 기억나지 않는다면 서버 쪽에서 비밀번호를 초기화할 수 있습니다.

```bash
npm run user -- passwd --username 내이름
```

이 명령은 비밀번호를 임시값으로 재설정합니다. 재설정 후 로그인하면 다시 비밀번호 변경 화면이 나타납니다.

> **주의:** 이 명령은 해당 계정의 모든 로그인 세션을 즉시 무효화합니다.

### 로그인 후 화면이 멈추거나 아무 반응이 없다

로그인 직후 서버에서 초기 데이터를 받아오는 데 잠깐 시간이 걸릴 수 있습니다 (네트워크 상태에 따라 수 초). 잠시 기다려 보세요.

그래도 화면이 완전히 굳어서 입력이 전혀 되지 않는다면, `Ctrl + C` 로 종료한 뒤 다시 실행하세요.

---

## 현재 제한 사항

- 세션이 만료되거나 서버 연결이 끊어졌을 때 자동으로 재로그인 화면이 나타나지 않습니다. 이런 상황이 발생하면 `Ctrl + C` 로 종료한 뒤 `npm run start:cli` 를 다시 실행해야 합니다.
- 비밀번호 입력 시 백스페이스를 여러 번 누르면 화면이 일시적으로 어색하게 보일 수 있습니다. 입력 자체는 정상 처리되므로 그냥 계속 진행하면 됩니다.
