## 1. 메시지 보내기

**무엇을 하는가** — 화면 아래 입력창에 질문이나 요청을 입력하고 presence에 전달합니다.

**어떻게 하는가**

1. 화면 아래쪽 `>` 표시 옆 입력창을 찾습니다.
2. 키보드로 하고 싶은 말을 입력합니다.
3. 다 입력했으면 `Enter`를 누릅니다.

```
> 오늘 할 일 목록을 정리해 줘_
```

**결과는 무엇인가** — 메시지가 화면 위쪽에 배경이 강조된 형태로 표시되고, 잠시 후 presence의 응답이 그 아래에 나타납니다.

> **참고** — 응답이 생성되는 동안에는 입력창이 비활성 상태가 됩니다. 이때 입력창 안에 회색으로 현재 상황을 알리는 힌트 문구가 표시됩니다.
>
> | 상황 | 입력창에 표시되는 문구 |
> |------|----------------------|
> | 응답 생성 중 | `> [응답 대기 중 · ESC로 취소]` |
> | 승인 요청 중 | `> [승인이 필요합니다 · y / n]` |
> | 서버 연결 끊김 | `> [연결 끊김 · Ctrl+C로 재시작]` |
>
> 이 문구가 보이면 지금 입력을 받을 수 없는 상태입니다. 응답이 끝나거나 연결이 복구될 때까지 기다리세요. 응답 생성 중이라면 `Esc`로 취소할 수도 있습니다.

## 2. 이전에 입력한 내용 다시 불러오기

**무엇을 하는가** — 이전에 보낸 메시지를 다시 입력하지 않고 빠르게 불러옵니다. 비슷한 질문을 반복하거나, 실수로 잘못 보낸 문장을 고쳐 보낼 때 유용합니다.

**어떻게 하는가**

- 입력창에서 `↑` (위 방향키)를 누르면 가장 최근에 보낸 메시지가 나타납니다.
- `↑`를 계속 누르면 그 이전 메시지들이 차례로 나옵니다. 최대 50개까지 불러올 수 있습니다.
- `↓` (아래 방향키)를 누르면 더 최근 메시지 방향으로 이동합니다.
- 처음 입력하던 내용으로 돌아가려면 `↓`를 계속 눌러 가장 아래까지 이동합니다.

**결과는 무엇인가** — 선택한 이전 메시지가 입력창에 채워집니다. 그대로 보낼 수도 있고, 일부를 수정한 뒤 `Enter`로 보낼 수도 있습니다.

> **참고** — 히스토리에서 불러온 내용을 수정하다가 `↑`/`↓`를 다시 누르면 수정 중이던 내용은 잠시 보관됩니다. `↓`를 눌러 맨 아래로 내려오면 수정하던 내용이 복원됩니다.

## 3. 응답이 만들어지는 동안 화면은 어떻게 보이나요?

**무엇을 하는가** — presence가 응답을 생성할 때 진행 상황을 화면에서 확인할 수 있습니다.

**어떻게 하는가** — 메시지를 보낸 뒤 화면 중간을 지켜보세요. 두 단계로 나뉩니다.

**1단계 — 생각 중**

```
thinking...
```

presence가 어떻게 답할지 처리하는 단계입니다. 이 메시지가 잠깐 표시됩니다.

**2단계 — 응답 작성 중**

```
안녕하세요! 오늘 할 일 목록을 정리해 드릴게요.▌
```

텍스트 끝에 깜빡이는 `▌` 표시가 붙습니다. 응답이 실시간으로 화면에 쌓이고 있다는 뜻입니다.

**결과는 무엇인가** — `▌`가 사라지고 입력창이 다시 활성화되면 응답이 완성된 것입니다.

> **참고** — 이전 버전에서는 응답을 받는 도중에 `receiving 42 chars...` 같은 내부 문구가 잠깐 보이기도 했습니다. 지금은 그런 문구 없이 `thinking...` 이후 바로 응답이 표시됩니다.

---

## 4. 응답 중간에 멈추고 싶을 때

**무엇을 하는가** — 응답이 너무 길거나, 잘못된 방향으로 가고 있을 때 중간에 멈춥니다.

**어떻게 하는가** — 응답이 생성되는 도중 `Esc` 키를 누릅니다.

**결과는 무엇인가** — 응답 생성이 중단되고 대화 창에 아래와 같은 시스템 메시지가 한 줄 나타납니다.

```
작업이 취소되었습니다
```

이 메시지로 취소가 실제로 적용되었음을 확인할 수 있습니다. 이미 화면에 나온 내용은 그대로 남아 있습니다. 중단된 지점까지의 내용을 확인한 뒤 다시 질문할 수 있습니다.

> **참고** — `Esc`는 응답이 만들어지는 중일 때만 취소로 작동합니다. 에이전트가 대기 중일 때 `Esc`를 누르면 화면에 잠깐 표시된 안내 메시지(회색 글씨)가 지워집니다.

## 5. 응답 읽기 — 굵은 글씨와 코드 강조

**무엇을 하는가** — presence의 응답에는 글씨 강조와 코드 표시가 포함될 수 있습니다.

**지원되는 표시 방식**

| 화면에 보이는 것 | 설명 |
|-----------------|------|
| **굵은 글씨** | 중요한 단어나 핵심 내용 강조 |
| `이런 형태의 글씨` (청록색) | 명령어, 파일명, 짧은 코드 조각 |
| 들여쓰기된 코드 블록 | 여러 줄짜리 코드나 출력 결과 |
| **제목 (굵고 흰색)** | `#`, `##`, `###` 으로 시작하는 섹션 제목 |
| 가로 구분선 | 섹션 사이를 구분하는 선 |

**현재 화면에서 그대로 보이는 기호들**

presence 화면은 터미널 환경의 특성상, 아래 표시 방식은 아직 지원되지 않습니다. 해당 기호가 그대로 표시될 수 있습니다.

| 원래 의도 | 화면에 보이는 것 | 읽는 방법 |
|----------|----------------|----------|
| 목록 항목 | `- 항목 내용` 또는 `• 항목 내용` | `-` 또는 `*` 다음 내용이 목록 항목입니다 |
| 기울임 글씨 | `_단어_` 또는 `*단어*` | 언더바나 별표 사이 단어가 강조된 내용입니다 |
| 링크 | `[링크 텍스트](주소)` | 대괄호 안 텍스트가 링크 이름입니다 |

> **팁** — 응답에 목록이 보이는데 `-` 기호가 그대로 붙어 있다면 "목록 항목"으로 읽으면 됩니다. 내용 이해에는 문제가 없습니다.

## 6. 이전 대화 전체 보기 (트랜스크립트)

**무엇을 하는가** — 현재 대화의 내부 처리 과정과 상세 내역을 별도 화면에서 확인합니다. 응답이 어떤 과정으로 만들어졌는지, 어떤 작업이 실행됐는지 파악할 때 유용합니다.

**어떻게 하는가** — 입력창이 활성화된 상태에서 `Ctrl+T`를 누릅니다.

**결과는 무엇인가** — 전체 화면이 트랜스크립트 보기로 전환됩니다.

```
[ 작업 흐름 ]  [ 대화 ]  [ 요청 내용 ]  [ 응답 내용 ]
```

화면 위쪽에 네 개의 탭이 표시됩니다.

**탭 안에서 이동하는 방법**

| 키 | 동작 |
|----|------|
| `←` / `→` | 탭 전환 |
| `↑` / `↓` | 현재 탭 내용 한 줄씩 스크롤 |
| `Page Up` / `Page Down` | 현재 탭 내용 한 페이지씩 스크롤 |
| `Ctrl+A` | 현재 탭 맨 위로 이동 |
| `Ctrl+E` | 현재 탭 맨 아래로 이동 |
| `Ctrl+O` | 작업 흐름 탭에서 상세 보기 켜기/끄기 |
| `Esc` 또는 `Ctrl+T` | 트랜스크립트 닫고 채팅으로 돌아가기 |

> **참고** — 트랜스크립트 화면이 열려 있는 동안에는 채팅 입력이 비활성화됩니다. 채팅으로 돌아가려면 반드시 `Esc`를 눌러 닫아야 합니다.

> **팁** — 채팅 화면 하단의 힌트 줄(`/help 커맨드 · Ctrl+T 전사 · Ctrl+O 도구 상세`)에서 `Ctrl+T` 진입 방법을 바로 확인할 수 있습니다. 전체 커맨드 목록이 궁금하면 `/help`를 입력하세요. 슬래시 커맨드에 대한 자세한 내용은 [04-slash-commands.md](04-slash-commands.md)를 참조하세요.

## 7. 화면에서 사라진 오래된 메시지 찾기

**무엇을 하는가** — 대화가 길어지면 오래된 메시지가 화면 위쪽에서 밀려납니다. 화면에 한 번에 표시되는 메시지는 최대 50개입니다.

**언제 이런 상황이 생기나요?** — 대화를 오랫동안 이어가다 보면 초반에 주고받은 내용이 화면 위로 사라집니다. 스크롤을 올려봐도 더 이상 올라가지 않는 느낌이 든다면, 50개 제한에 도달한 것입니다.

**찾는 방법** — `Ctrl+T`를 눌러 트랜스크립트를 열면 화면에서 사라진 내용을 포함한 대화 흐름 전체를 확인할 수 있습니다.

**결과는 무엇인가** — 트랜스크립트의 `[ 대화 ]` 탭에서 처음부터의 대화 내역을 스크롤하며 확인할 수 있습니다.

> **참고** — 화면 표시 한도(50개)와 실제 대화 기억은 별개입니다. 화면에서 보이지 않더라도 presence는 대화 맥락을 유지하고 있습니다. 오래된 내용을 직접 찾아보고 싶을 때만 트랜스크립트를 활용하세요.

## 8. 대화 내용을 파일로 저장하거나 복사하기

### 응답을 클립보드에 바로 복사하기 (`/copy`)

**무엇을 하는가** — 가장 최근에 받은 에이전트 응답을 클립보드에 복사합니다. 다른 앱에 바로 붙여넣을 수 있습니다.

**어떻게 하는가** — 입력창에 `/copy`를 입력하고 `Enter`를 누릅니다.

```
> /copy
마지막 응답이 클립보드에 복사되었습니다
```

복사할 응답이 없으면 `복사할 응답이 없습니다` 라고 표시됩니다.

> **팁** — 코드 블록이나 URL이 포함된 응답을 빠르게 복붙할 때 가장 편리합니다. 화면에서 텍스트를 마우스로 선택해 복사하는 방식은 터미널 환경의 특성상 지원되지 않으므로, `/copy`를 활용하세요.

### 대화 요약을 파일로 저장하기 (`/report`)

**무엇을 하는가** — 현재 대화의 요약 정보를 파일로 저장합니다. 대화를 마친 뒤 내용을 보관하거나, 관리자에게 문제 상황을 전달할 때 사용합니다.

**어떻게 하는가** — 입력창에 `/report`를 입력하고 `Enter`를 누릅니다.

```
> /report
```

**결과는 무엇인가** — 대화 요약이 아래 위치에 파일로 저장됩니다.

```
~/.presence/reports/report-2026-04-25T10-00-00.md
```

파일 저장이 완료되면 화면에 저장된 경로와 함께 결과가 표시됩니다.

```
리포트 저장됨: /Users/사용자이름/.presence/reports/report-2026-04-25T10-00-00.md
(클립보드에 복사됨)
```

macOS에서는 파일 저장과 동시에 내용이 클립보드에도 복사됩니다.

> **참고** — `/report`로 저장되는 내용은 주고받은 대화 메시지가 아니라, 마지막 처리 과정의 기술적 요약(요청 내용, 작업 단계, 모델 설정 등)입니다. 순수하게 대화 내용만 보관하고 싶다면 트랜스크립트(`Ctrl+T`)에서 직접 확인하세요.

## 9. 대화창에 시스템 메시지가 나타날 때

대화 메시지가 아닌 회색 또는 별도 스타일로 표시되는 시스템 메시지가 대화창에 나타날 수 있습니다. 주요 유형을 알아둡시다.

### 작업 취소 메시지

```
작업이 취소되었습니다
```

`Esc`를 눌러 응답을 취소하면 이 메시지가 나타납니다. 정상 동작입니다.

### "[A2A 응답 from ...]" 메시지

```
[A2A 응답 from alice/default] (작업 결과 내용)
[A2A 응답 실패 from alice/default] 수신자 메시지 대기열이 가득 찼습니다
```

에이전트가 복잡한 작업을 처리할 때 내부적으로 세부 작업을 다른 에이전트에게 나눠줄 수 있습니다. 그 결과가 이런 형태로 대화창에 표시됩니다.

- `[A2A 응답 from ...]` — 위임된 작업이 완료된 결과입니다. 내용을 확인하고 계속 대화하면 됩니다.
- `[A2A 응답 실패 from ...]` — 위임 작업이 실패했습니다. 요청을 다시 입력해 보세요.
- `[A2A 응답 타임아웃 from ...]` — 위임 작업이 시간 초과됐습니다. 잠시 후 다시 시도하세요.

> **참고** — `alice/default` 형태의 식별자가 보이는 것은 현재 알려진 표시 방식입니다(개선 예정). 유저 측에서 바꿀 수 없으며, 향후 업데이트에서 더 친숙한 표현으로 바뀔 예정입니다.

더 자세한 시스템 메시지 안내는 [05-troubleshooting.md](05-troubleshooting.md)를 참고하세요.

## 문제가 생기면

| 증상 | 확인할 것 |
|------|-----------|
| 글자를 눌러도 입력이 안 됩니다 | 입력창 안에 회색 힌트 문구가 보이는지 확인하세요. `[응답 대기 중 · ESC로 취소]`가 보이면 응답이 끝날 때까지 기다리거나 `Esc`로 취소하세요. `[연결 끊김 · Ctrl+C로 재시작]`이 보이면 TUI를 재시작해야 합니다. |
| 트랜스크립트를 닫으려는데 아무 반응이 없습니다 | 트랜스크립트가 열려 있는 상태에서 `Esc`를 누르세요. |
| 오래된 메시지가 보이지 않습니다 | `Ctrl+T`로 트랜스크립트를 열어 `[ 대화 ]` 탭을 확인하세요. |
| `/report` 저장 경로를 모르겠습니다 | 저장 후 화면에 전체 경로가 표시됩니다. |
| `/copy`를 입력했는데 아무것도 안 됩니다 | 아직 에이전트 응답이 없는 경우입니다. 먼저 메시지를 보내 응답을 받은 뒤 사용하세요. |
