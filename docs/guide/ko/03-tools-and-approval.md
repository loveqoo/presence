# 03. 도구 실행과 승인

에이전트가 여러분의 요청을 처리할 때는 혼자 생각하는 것 외에도 실제로 무언가를 합니다. 파일을 읽거나, 명령어를 실행하거나, 웹에서 정보를 가져오는 것처럼요. 이 문서는 그런 "도구 실행"이 화면에서 어떻게 보이는지, 그리고 위험한 작업이 있을 때 여러분에게 허락을 구하는 방법을 설명합니다.

## 1. 에이전트가 사용하는 도구

"도구"란 에이전트가 화면 밖에서 실제로 수행하는 행동을 말합니다.

| 도구 이름 | 하는 일 | 예시 |
|-----------|---------|------|
| `file_read` | 파일을 읽어 내용을 확인합니다 | 코드 파일이나 문서 내용 파악 |
| `file_write` | 파일에 내용을 씁니다 | 메모 저장, 코드 수정 |
| `file_list` | 폴더 안의 파일 목록을 봅니다 | 어떤 파일이 있는지 파악 |
| `shell_exec` | 터미널 명령어를 실행합니다 | 프로그램 설치, 빌드 실행 등 |
| `web_fetch` | 인터넷 주소에서 내용을 가져옵니다 | 웹 페이지나 API 호출 |
| `calculate` | 수식을 계산합니다 | 숫자 계산 |

이 외에도 관리자가 추가로 연결한 도구(MCP 도구)가 있을 수 있습니다. 연결된 도구 전체 목록은 `/tools`를 입력하면 확인할 수 있습니다.

> **도구 목록 보기**
> 입력창에 `/tools` 를 입력하고 Enter를 누릅니다.
> 화면에 현재 사용 가능한 도구 이름 목록이 표시됩니다.

## 2. 상태 표시줄로 현재 상황 파악하기

화면 맨 아래에 한 줄짜리 **상태 표시줄**이 있습니다. 에이전트가 지금 무엇을 하고 있는지 한눈에 볼 수 있습니다.

### 상태 표시줄이 보이는 위치

```
┌────────────────────────────────────────────────────────────────┐
│ 대화 내용이 여기에 표시됩니다                                   │
│                                                                │
│ ...                                                            │
├────────────────────────────────────────────────────────────────┤
│ > 여기에 메시지를 입력합니다                                   │
├────────────────────────────────────────────────────────────────┤
│ ● idle │ budget: 12% │ gpt-4o │ presence │ main              │
└────────────────────────────────────────────────────────────────┘
```

맨 아래 줄이 상태 표시줄입니다.

### 에이전트 상태 아이콘

상태 표시줄 맨 왼쪽에 현재 상태가 표시됩니다.

| 화면에 보이는 것 | 의미 |
|-----------------|------|
| `● idle` (초록색) | 에이전트가 대기 중입니다. 메시지를 입력할 수 있습니다. |
| `⠋ thinking... 3s` (노란색, 스피너 회전) | 에이전트가 생각하거나 작업 중입니다. 잠시 기다려 주세요. |
| `✗ error` (빨간색) | 작업 중 문제가 발생했습니다. 아래 [에러 확인 방법](#9-에러가-발생했을-때)을 참고하세요. |

작업 중일 때는 스피너 옆에 현재 진행 내용(예: `thinking...`)과 경과 시간(예: `3s`, `1m20s`)이 함께 표시됩니다.

### 상태 표시줄에 표시되는 추가 정보

상태 표시줄에는 `│` 구분자로 구분된 여러 정보가 함께 표시될 수 있습니다.

| 표시 항목 | 의미 |
|-----------|------|
| `budget: 45%` | 이번 대화에서 사용한 처리량 비율 (80% 이상이면 노란색, 95% 이상이면 빨간색으로 변합니다) |
| `gpt-4o` | 현재 사용 중인 AI 모델 이름 |
| `presence` | 현재 작업 디렉토리 이름 |
| `main` | 현재 Git 브랜치 이름 |
| `tools: 12` | 사용 가능한 도구 수 |
| `mem: 8` | 에이전트가 기억하고 있는 항목 수 |
| `turn: 5` | 현재까지 주고받은 대화 횟수 |

표시 항목은 `/statusline` 커맨드로 켜고 끌 수 있습니다. 자세한 내용은 [슬래시 커맨드 가이드](./04-slash-commands.md)를 참고하세요.

## 3. 사이드패널 열기

화면 오른쪽에 **사이드패널**을 열면 에이전트의 현재 상태를 한눈에 볼 수 있습니다.

> **사이드패널 열고 닫기**
> 입력창에 `/panel` 을 입력하고 Enter를 누릅니다.
> 다시 `/panel` 을 입력하면 닫힙니다.

사이드패널이 열리면 화면 오른쪽에 이런 내용이 표시됩니다:

```
│── Agents ──
│  ● presence
│
│── Tools ──
│  file_read
│  file_write
│  shell_exec
│  web_fetch
│  calculate
│  +3 more
│
│── Memory ──
│  12 nodes
│
│── TODOs ──
│  보고서 작성
│  이메일 확인
│
│── Events ──
│  queue empty
```

### 각 섹션 설명

**Agents** — 현재 실행 중인 에이전트 목록입니다.

**Tools** — 에이전트가 사용할 수 있는 도구 목록입니다. 8개까지 표시되며 더 있으면 `+N more`로 표시됩니다. 전체 목록은 `/tools`로 확인하세요.

**Memory** — 에이전트가 기억하고 있는 항목 수입니다. 기억을 조회하거나 정리하려면 `/memory` 슬래시 커맨드를 사용하세요 ([04-slash-commands.md](./04-slash-commands.md) 참고).

**TODOs** — 에이전트가 파악한 할 일 목록입니다. 5개까지 표시되며 더 있으면 `+N more`로 표시됩니다. 전체 목록은 `/todos`로 확인하세요.

**Events** — 외부에서 들어온 이벤트 대기 현황입니다. 평소에는 `queue empty`로 표시됩니다. 이벤트가 처리 실패하면 `/status`로 상세 내용을 확인하세요.

## 4. 도구 실행 결과 읽기 — 접기/펼치기

에이전트가 도구를 실행하면 그 결과가 대화 창에 표시됩니다. 기본적으로는 한 줄 요약만 보입니다.

### 접힌 상태 (기본)

```
tool   > file_read /home/user/report.txt — 42 lines
tool   > $ npm install — 8 lines
tool   > web_fetch https://example.com — 15 lines
```

`tool   >` 뒤에 도구 이름과 간단한 요약이 보입니다.

### 펼쳐서 전체 내용 보기

**Ctrl+O** 를 누르면 모든 도구 결과가 펼쳐집니다. 다시 **Ctrl+O** 를 누르면 다시 접힙니다.

펼친 상태에서는 파일 내용, 명령어 실행 결과, 웹 내용 등이 자세히 보입니다.

```
tool   > file_read /home/user/report.txt
   1  # 월간 보고서
   2
   3  ## 요약
   4  이번 달 매출은 ...
   ...
```

> **참고**: 현재 접힌/펼친 상태를 화면에서 바로 알기 어려울 수 있습니다. 도구 결과가 한 줄로만 보인다면 접힌 상태입니다. Ctrl+O로 전환해 보세요.

## 5. 플랜 뷰 — 에이전트의 작업 계획 보기

여러 단계가 필요한 작업을 할 때, 에이전트가 각 단계를 어떻게 진행하는지 보여주는 **플랜 뷰**가 대화 창에 나타납니다.

```
  ── iteration 1/3 ────────────────────────
  ├─ ✓ 파일 읽기: /home/user/data.csv
  ├─ ◐ AI 분석 중...
  └─ ○ 결과 응답 준비
```

각 줄 앞의 아이콘은 단계 상태를 나타냅니다.

| 아이콘 | 의미 |
|--------|------|
| `○` | 아직 시작 안 함 |
| `◐` | 진행 중 |
| `✓` | 완료 |
| `✗` | 오류 발생 |
| `⚠` | 승인 대기 중 |

### Op 코드에 대하여

플랜 뷰에는 가끔 내부 코드 이름이 보일 수 있습니다. 아래 표를 참고해 의미를 파악하세요.

| 화면에 보이는 코드 | 실제 의미 |
|-------------------|-----------|
| `EXEC file_read(...)` | 파일 읽기 실행 |
| `EXEC shell_exec(...)` | 터미널 명령어 실행 |
| `ASK_LLM "..."` | AI에게 분석 요청 |
| `LOOKUP_MEMORY "..."` | 저장된 기억 검색 |
| `DELEGATE → agent_name` | 다른 에이전트에게 작업 위임 |
| `RESPOND message` | 대화 창에 응답 전송 |
| `APPROVE: ...` | 승인 요청 (아래 참고) |

> 이 코드 이름들은 시스템 내부 표현입니다. 앞으로 더 알기 쉬운 한국어로 개선될 예정입니다.

## 6. 코드 뷰 — 파일 내용 보기

에이전트가 파일을 읽거나 코드를 보여줄 때는 줄 번호와 색상 강조가 적용된 **코드 뷰**로 표시됩니다.

```
  1  const greeting = 'Hello, world!'
  2  console.log(greeting)
  3  // 이 줄은 주석입니다
```

### 80줄 제한

파일이 80줄보다 길면 처음 80줄만 표시되고 나머지는 이렇게 잘립니다.

```
 78  ...
 79  ...
 80  ...
    ... +120 lines
```

**잘린 내용을 보고 싶다면:**

현재 TUI 화면에서는 80줄 이후 내용을 직접 스크롤할 수 없습니다. 이런 경우 에이전트에게 직접 물어보세요.

```
특정 부분이 궁금하면 이렇게 말하세요:
"report.txt 파일의 100번째 줄부터 150번째 줄 사이 내용을 보여줘"
또는
"report.txt 파일에서 '오류' 라는 단어가 있는 부분만 찾아줘"
```

에이전트가 해당 부분만 가져와 대화 창에 표시해 줍니다.

## 7. 승인 프롬프트 — 허락을 구할 때

에이전트가 돌이키기 어렵거나 위험할 수 있는 작업을 하기 전에는 반드시 **여러분의 허락**을 먼저 구합니다.

### 승인 프롬프트가 뜨는 순간

```
⚠ APPROVE: shell_exec: rm -rf ./temp_files
  [y] approve  [n] reject
```

입력창이 잠기고 위와 같은 노란색 경고 메시지가 나타납니다.

### 어떻게 답하는가

| 키 | 동작 |
|----|------|
| `y` 또는 `Y` | 허락합니다. 에이전트가 해당 작업을 실행합니다. |
| `n` 또는 `N` | 거부합니다. 해당 작업을 실행하지 않고 중단합니다. |

승인 프롬프트가 떠 있는 동안은 다른 메시지를 입력할 수 없습니다. 반드시 `y` 또는 `n`으로 답해야 합니다.

### 거부했을 때 일어나는 일

`n`을 누르면 해당 작업이 취소되고 에이전트가 멈춥니다. 화면에서 승인 프롬프트가 사라집니다.

> **주의**: 현재 버전에서는 거부 후 "거부되었습니다"라는 별도 메시지가 표시되지 않을 수 있습니다. 프롬프트가 사라졌다면 거부가 정상적으로 처리된 것입니다. 확인하려면 `/status`를 입력해 현재 상태를 확인하세요.

### 위험한 작업인지 판단하는 방법

> **⚠ 중요 안내 — 반드시 description 텍스트를 읽으세요**
>
> 현재 버전에서는 모든 승인 요청이 동일한 노란색 경고 형식으로 표시됩니다. 파일을 읽는 것처럼 안전한 작업과 파일을 삭제하거나 명령어를 실행하는 위험한 작업이 같은 모습으로 표시됩니다.
>
> 따라서 **`⚠ APPROVE:` 뒤에 오는 설명 텍스트를 반드시 직접 읽어야 합니다.**
>
> 아래 단어가 보이면 특히 주의하세요.
>
> | 설명에 이 단어가 보이면 | 특히 주의 필요 |
> |------------------------|--------------|
> | `shell_exec`, `rm`, `delete`, `drop` | 명령어 실행 또는 삭제 작업 |
> | `file_write` + 중요한 경로 | 기존 파일 덮어쓰기 가능 |
> | `--force`, `-f`, `-rf` | 강제 실행 플래그 |
>
> **모르는 내용이 포함되어 있으면 `n`으로 거부하고 에이전트에게 설명을 먼저 요청하세요.**

## 8. 작업 취소하기 (Esc)

에이전트가 작업 중일 때 **Esc** 키를 누르면 진행 중인 작업을 취소할 수 있습니다.

### Esc 키의 두 가지 동작

**에이전트가 작업 중일 때 (상태 표시줄에 스피너가 돌고 있을 때)**

`Esc` → 현재 작업이 즉시 취소됩니다. 에이전트가 멈추고 상태가 `idle`로 돌아옵니다.

**에이전트가 대기 중일 때 (상태 표시줄이 `● idle`일 때)**

`Esc` → 화면에 표시된 임시 안내 메시지가 사라집니다. 대화 내용에는 영향이 없습니다.

> **주의**: 작업 중에 실수로 Esc를 누르면 작업이 취소됩니다. 현재 버전에서는 취소 후 별도 확인 메시지가 표시되지 않을 수 있습니다. 작업 상태가 궁금하면 `/status`로 확인하세요.

## 9. 에러가 발생했을 때

상태 표시줄에 `✗ error`(빨간색)가 표시되면 마지막 작업에서 문제가 생긴 것입니다.

### 원인 확인 방법 1 — `/status`

입력창에 `/status`를 입력하고 Enter를 누릅니다.

```
status: error | turn: 5 | mem: 12 | last: failure
```

`last: failure`는 마지막 작업이 실패했다는 의미입니다.

### 원인 확인 방법 2 — `/report`

더 자세한 원인이 필요하다면 `/report`를 입력합니다.

```
/report
```

에이전트의 마지막 작업에 대한 상세 보고서가 생성되어 `~/.presence/reports/` 폴더에 저장됩니다. 맥(Mac)에서는 클립보드에도 자동으로 복사됩니다.

### 자주 있는 에러 상황

| 상황 | 해결 방법 |
|------|----------|
| 에이전트가 갑자기 멈추고 `✗ error` 표시 | `/status`로 현재 상태 확인 후 다시 메시지를 보내 재시도 |
| 응답이 너무 오래 걸림 | Esc로 취소 후 더 간단하게 요청을 나눠서 다시 시도 |
| `✗ error`가 반복해서 나타남 | `/report`로 보고서를 만들어 관리자에게 전달 |

> **에러 후 재시도**: `✗ error` 상태에서도 새 메시지를 입력해 작업을 다시 시작할 수 있습니다. 에이전트는 새 메시지를 받으면 상태가 자동으로 초기화됩니다.
