## 1. 대화창에 낯선 시스템 메시지가 나타납니다

### "서버가 재시작되어 처리 중이던 메시지가 중단되었습니다"

```
서버가 재시작되어 처리 중이던 메시지가 중단되었습니다
```

presence의 에이전트는 복잡한 작업을 처리할 때 내부적으로 다른 에이전트에게 세부 작업을 넘기기도 합니다. 이 과정 도중 서버가 재시작되면 전달 중이던 작업이 중단되며 위와 같은 메시지가 대화창에 표시됩니다.

**어떻게 해야 하나요?** — 이 메시지는 알림이며, 일반 대화에는 영향이 없습니다. 메시지가 보인 후에도 평소처럼 대화를 계속하면 됩니다. 직전에 요청한 작업이 제대로 완료되지 않았을 수 있으니, 필요하면 다시 같은 요청을 입력하세요.

관련 메시지들:

| 대화창에 표시되는 내용 | 의미 |
|----------------------|------|
| `서버가 재시작되어 처리 중이던 메시지가 중단되었습니다` | 작업 전달 중 서버 재시작으로 중단됨 |
| `서버 재시작 후 수신 에이전트가 등록되어 있지 않습니다` | 재시작 후 수신 측 에이전트가 아직 준비되지 않음 |
| `서버 재시작 후 수신 큐 재진입에 실패했습니다` | 재시작 후 작업 재시도 중 문제 발생 |

모두 서버 재시작 직후 일시적으로 나타나는 메시지입니다.

### "수신자 메시지 대기열이 가득 찼습니다"

```
수신자 메시지 대기열이 가득 찼습니다
```

에이전트가 내부 작업을 넘기려는 상대방이 현재 처리할 여유가 없을 때 나타납니다.

**어떻게 해야 하나요?** — 잠시 기다린 뒤 다시 같은 요청을 입력해 보세요. 반복된다면 서버 관리자에게 문의하세요.

### "[A2A 응답 from ...]" 형태의 메시지

```
[A2A 응답 from alice/default] (작업 결과 내용)
[A2A 응답 실패 from alice/default] 수신자 메시지 대기열이 가득 찼습니다
[A2A 응답 타임아웃 from alice/default]
```

에이전트가 세부 작업을 처리한 결과가 대화창에 시스템 메시지 형태로 표시될 수 있습니다. `alice/default` 부분은 작업을 처리한 내부 에이전트의 식별자입니다.

> **참고** — 이 메시지 형식은 현재 개선 예정입니다(FP-67). 지금은 내부 식별자("A2A", `alice/default` 같은 형태)가 그대로 노출되고 있습니다. 향후 업데이트에서 더 친숙한 표현으로 바뀔 예정입니다.

**어떻게 해야 하나요?** — `[A2A 응답 from ...]` 메시지가 보이면 에이전트가 위임한 작업 결과입니다. 내용을 확인하고 계속 대화하면 됩니다. `실패` 또는 `타임아웃`이 포함된 경우 해당 작업이 완료되지 않았다는 의미이며, 요청을 다시 입력해 보세요.

## 2. 상태바에 에러가 표시됩니다

화면 맨 아래 상태바에 다음과 같은 표시가 나타나면 직전 응답 중 문제가 생긴 것입니다.

```
✗ error: interpreter
```

| 표시 | 의미 |
|------|------|
| `error: interpreter` | 도구(파일 읽기, 명령 실행 등) 실행 중 문제 발생 |
| `error: planner_parse` | AI 응답을 처리하는 중 문제 발생 |
| `error: planner_shape` | AI 응답 형식이 예상과 다름 |
| `error: max_iterations` | 에이전트가 너무 많은 단계를 시도함 |

**어떻게 해야 하나요?**

1. 같은 질문을 다시 입력해 보세요. 일시적인 문제라면 다음 시도에 정상 처리됩니다.
2. 질문을 더 구체적이고 짧게 나눠서 다시 시도해 보세요.
3. `/report`를 입력하면 상세 내용이 파일로 저장됩니다. 문제가 반복된다면 이 파일을 관리자에게 전달하세요.

> **에러 후 재시도** — `✗ error` 상태에서도 새 메시지를 입력하면 에이전트가 자동으로 다시 시작합니다. 에러 상태가 계속 이어지지 않으니 당황하지 마세요.

## 3. 예산 경고가 뜹니다

대화창에 이런 경고가 나타나면 대화 이력이 너무 길어진 것입니다.

```
⚠ 프롬프트 예산 85% — 대화 이력 3개가 제외되었습니다. /clear로 이력을 정리하세요.
```

또는:

```
⚠ 프롬프트 예산 92% — /clear 또는 /memory clear로 컨텍스트를 정리하세요.
```

AI가 한 번에 처리할 수 있는 대화 분량에는 한계가 있습니다. 이 한계(예산)에 가까워지면 경고가 표시됩니다.

**어떻게 해야 하나요?**

- `/clear` — 화면의 대화 이력을 지웁니다. 에이전트가 기억하고 있는 내용(기억 노드)은 지워지지 않습니다.
- `/memory clear 7d` — 7일 이상 된 기억을 지워 예산을 확보합니다.
- 두 가지를 함께 사용하면 예산을 가장 많이 확보할 수 있습니다.

> **주의** — `/clear`로 지운 대화 이력은 복구할 수 없습니다. 보관이 필요하면 `/report`로 먼저 저장하세요.

상태바의 `budget: 85%` 항목이 **노란색**으로 바뀌면 80% 이상, **빨간색**으로 바뀌면 95% 이상 사용된 것입니다.

## 4. 연결이 끊겼을 때

대화 도중 화면 상단에 빨간 배너가 나타나면 서버 연결이 끊긴 것입니다.

```
세션이 만료되었습니다 (close 4001)
TUI 를 재시작하세요 (Ctrl+C).
```

입력창에도 `[연결 끊김 · Ctrl+C로 재시작]` 문구가 표시되며 입력이 막힙니다.

**배너 문구별 의미와 해결 방법:**

| 배너 문구 | 의미 | 해결 |
|-----------|------|------|
| `세션이 만료되었습니다 (close 4001)` | 로그인 토큰이 만료됨 | `Ctrl+C` 후 다시 `npm run start:cli` 실행하여 재로그인 |
| `비밀번호 변경이 필요합니다 (close 4002)` | 관리자가 비밀번호를 재설정함 | `Ctrl+C` 후 재시작 → 로그인 후 비밀번호 변경 화면이 자동으로 나타남 |
| `접근이 거부되었습니다 (close 4003)` | 허용되지 않은 접속 | 서버 관리자에게 문의 |
| `서버 연결이 끊겼습니다 (close 4004)` | 실행 폴더가 허용 범위 밖 | 아래 [5. 허용되지 않는 폴더](#5-허용되지-않는-폴더에서-실행했을-때) 참고 |
| `서버 연결이 끊겼습니다 (close 1006)` | 네트워크 단절 또는 서버 종료 | 서버가 실행 중인지 확인 후 `Ctrl+C` → 재시작 |

모든 경우에 `Ctrl+C`로 TUI를 종료한 뒤 원인을 조치하고 `npm run start:cli`를 다시 실행합니다.

## 5. 허용되지 않는 폴더에서 실행했을 때

TUI를 실행했을 때 또는 세션을 만들 때 아래와 같은 오류가 나타날 수 있습니다.

**빨간 배너로 앱 전체가 막히는 경우:**

```
⚠ 작업 디렉토리가 허용 범위 밖입니다 (close 4004).
TUI 를 재시작하세요 (Ctrl+C).
```

**세션 커맨드 실행 시 오류 메시지가 나오는 경우:**

```
오류: 현재 폴더가 서버의 허용 범위를 벗어났습니다. 허용된 폴더에서 다시 시도하세요.
```

두 경우 모두, TUI를 실행한 터미널의 현재 폴더가 서버에서 허용하는 범위 밖입니다.

**해결 방법:**

1. `Ctrl+C`로 TUI를 종료합니다.
2. 서버가 처음 켜졌던 폴더(또는 그 하위 폴더)로 이동합니다.
   ```bash
   cd /서버를-켰던-폴더
   ```
3. 다시 `npm run start:cli`를 실행합니다.

> **서버 허용 폴더를 바꾸고 싶다면** — `~/.presence/users/내이름/config.json` 파일에서 `tools.allowedDirs` 항목을 수정하세요. 서버 재시작 후 적용됩니다. 설정 방법을 모르면 서버 관리자에게 문의하세요.

## 6. 슬래시 커맨드가 작동하지 않습니다

### 오타 커맨드를 입력했을 때

`/`로 시작하지만 등록되지 않은 커맨드를 입력하면 에이전트로 넘어가지 않고 바로 안내 메시지가 표시됩니다.

```
알 수 없는 커맨드: /mem
/help 로 전체 목록 확인
```

올바른 커맨드를 확인하고 다시 입력하세요.

| 잘못 입력한 것 | 올바른 커맨드 |
|---------------|--------------|
| `/mem` | `/memory` |
| `/model` | `/models` |
| `/sessions` | `/session list` |
| `/tools` | `/tool list` |

### `/`로 시작하는 내용을 에이전트에게 전달하고 싶을 때

슬래시(`/`)로 시작하는 경로나 텍스트를 에이전트에게 직접 전달하려면, 문장 안에 포함해서 입력하세요.

```
/usr/local/bin 폴더를 확인해줘
```

이렇게 문장의 중간이나 끝에 오면 에이전트에게 그대로 전달됩니다.

## 7. 응답이 너무 오래 걸리거나 잘린 것 같습니다

### 응답 도중 상태바에 "재시도 중..." 이 표시됩니다

```
⠋ 재시도 2/3...
```

또는:

```
⠋ 재시도 2/3 — 응답 절단
```

AI가 응답을 생성하는 도중 일부 문제가 발생해 자동으로 다시 시도하고 있습니다. `응답 절단`이 표시되면 응답이 너무 길어서 잘린 경우입니다. 잠시 기다리면 자동으로 처리됩니다.

**어떻게 해야 하나요?** — 기다리거나, 너무 오래 걸린다면 `Esc`로 취소한 뒤 요청을 더 짧게 나눠서 다시 입력하세요.

### 응답이 길어도 너무 오래 기다려야 합니다

복잡한 작업(여러 파일 분석, 긴 계획 수립 등)은 기본적으로 시간이 걸립니다. 상태바의 스피너와 경과 시간을 보면서 진행 중인지 확인하세요.

```
⠋ thinking... 35s
```

`Esc`로 취소한 뒤 요청을 여러 개의 작은 단계로 나눠서 순서대로 요청하면 더 빠르게 처리될 수 있습니다.

## 8. 로그인 문제

### 비밀번호를 3번 틀렸을 때

TUI가 자동으로 종료됩니다. `npm run start:cli`를 다시 실행하고 로그인을 재시도하세요.

비밀번호가 기억나지 않는다면 서버 관리자에게 초기화를 요청하거나, 직접 서버에서 아래 명령을 실행할 수 있습니다.

```bash
npm run user -- passwd --username 내이름
```

임시 비밀번호로 재설정되며, 다시 로그인하면 새 비밀번호를 설정하는 화면이 자동으로 나타납니다.

> **주의** — 비밀번호 재설정 시 해당 계정의 모든 로그인 세션이 즉시 무효화됩니다. 이미 연결된 TUI 창이 있다면 끊깁니다.

### 로그인 화면이 나오지 않고 바로 종료됩니다

서버에 등록된 사용자가 없으면 서버가 시작을 거부합니다. 사용자 등록이 완료되었는지 확인하세요. ([01-getting-started.md](01-getting-started.md) 3단계 참고)

## 문제가 해결되지 않는다면

`/report`를 입력하면 현재 에이전트 상태와 직전 처리 내용이 파일로 저장됩니다. 이 파일을 서버 관리자에게 전달하면 원인 파악에 도움이 됩니다.

```
> /report
리포트 저장됨: /Users/사용자이름/.presence/reports/report-2026-04-25T10-00-00.md
(클립보드에 복사됨)
```
