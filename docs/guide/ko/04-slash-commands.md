> 최종 업데이트: 2026-04-11

# 슬래시 커맨드 레퍼런스

이 문서는 presence TUI에서 사용할 수 있는 모든 슬래시 커맨드를 설명합니다.

---

## 슬래시 커맨드란?

대화 입력창에 `/`(슬래시)로 시작하는 특수 명령을 입력하면 에이전트에게 말을 거는 대신 앱을 직접 제어할 수 있습니다.

예를 들어 `/status` 를 입력하면 현재 에이전트 상태를 화면에 표시하고, `/memory list` 를 입력하면 저장된 기억 목록을 볼 수 있습니다.

> **알아두세요** — 슬래시 커맨드가 아닌 일반 문장은 그대로 에이전트에게 전달됩니다. `/`로 시작하더라도 아래 목록에 없는 커맨드를 입력하면 **에이전트가 그 문장을 그대로 받아 대화를 시작합니다.** (예: `/무엇을 해야 할까요?` → 에이전트에 전달됨) 오타에 주의하세요. 자세한 내용은 [알려진 제한](#8-알려진-제한) 항목을 참고하세요.

---

## 1. 전체 커맨드 한눈에 보기

| 커맨드 | 한 줄 설명 | 결과가 화면에서 사라지나요? |
|--------|-----------|--------------------------|
| `/help` | 커맨드 목록 표시 | 예 (일시 표시) |
| `/status` | 현재 에이전트 상태 표시 | 예 (일시 표시) |
| `/tools` | 등록된 도구 목록 표시 | 예 (일시 표시) |
| `/todos` | 에이전트 할 일 목록 표시 | 예 (일시 표시) |
| `/models` | 사용 가능한 AI 모델 조회 및 전환 | 아니오 (영구 변경) |
| `/memory` | 저장된 기억 관리 | 예 (조회), 아니오 (삭제) |
| `/sessions` | 세션 목록 조회 및 관리 | 아니오 (영구 변경) |
| `/statusline` | 화면 아래 상태바 항목 추가/제거 | 아니오 (영구 변경) |
| `/mcp` | MCP 서버 조회 및 활성화/비활성화 | 아니오 |
| `/clear` | 대화 이력 초기화 | 아니오 (영구 변경) |
| `/panel` | 사이드 패널 열기/닫기 | 해당 없음 |
| `/report` | 디버그 리포트 파일로 저장 | 아니오 |
| `/quit` 또는 `/exit` | 앱 종료 | 해당 없음 |

> **"결과가 화면에서 사라지나요?"** — "예"인 커맨드는 조회 결과를 일시적으로 표시합니다. 다음 입력을 하거나 ESC 키를 누르면 자동으로 사라집니다. "아니오"인 커맨드는 설정이나 데이터를 영구 변경합니다.

## 2. 커맨드 상세 설명

### `/help` — 도움말 보기

```
/help
```

화면에 전체 커맨드 목록과 단축키를 표시합니다. 커맨드 이름이 기억나지 않을 때 가장 먼저 입력하세요.

---

### `/status` — 현재 상태 확인

```
/status
```

에이전트의 현재 상태를 한 줄로 표시합니다.

**표시 예시:**

```
status: idle | turn: 5 | mem: 12 | last: plan
```

| 항목 | 설명 |
|------|------|
| `status` | `idle`(대기 중) 또는 `working`(처리 중) |
| `turn` | 지금까지 진행한 대화 횟수 |
| `mem` | 저장된 기억 개수 |
| `last` | 직전 처리 결과 유형 |

---

### `/tools` — 도구 목록 확인

```
/tools
```

에이전트가 현재 사용할 수 있는 도구 이름 목록을 표시합니다.

---

### `/todos` — 할 일 목록 확인

```
/todos
```

에이전트가 현재 작업 중인 할 일 목록을 표시합니다. 항목이 없으면 `(none)` 이 표시됩니다.

---

### `/models` — AI 모델 조회 및 전환

```
/models                  ← 사용 가능한 모델 목록 조회
/models gpt-4o           ← 특정 모델로 전환
/models claude-3-5-haiku ← 다른 모델로 전환
```

서버에서 사용 가능한 AI 모델 목록을 조회하거나, 원하는 모델로 전환할 수 있습니다.

**조회 결과 예시:**

```
사용 가능한 모델:
  ● claude-3-5-sonnet (현재)
    gpt-4o
    claude-3-5-haiku
```

- `●` 표시가 있는 것이 현재 사용 중인 모델입니다.
- 모델 이름을 함께 입력하면 즉시 전환됩니다.

> **영구 변경** — 모델 전환은 현재 세션에 즉시 적용되며, 다음 대화부터 새 모델이 사용됩니다.

---

### `/clear` — 대화 이력 초기화

```
/clear
```

화면의 대화 이력을 모두 지웁니다. 저장된 **기억(memory)은 삭제되지 않습니다.**

> **영구 변경** — 삭제된 대화 이력은 되돌릴 수 없습니다. 프롬프트 용량이 가득 찼다는 경고가 뜰 때 사용하면 좋습니다.

---

### `/panel` — 사이드 패널 토글

```
/panel
```

화면 오른쪽 사이드 패널을 열거나 닫습니다. 입력할 때마다 열림/닫힘이 번갈아 적용됩니다.

---

### `/report` — 디버그 리포트 저장

```
/report
```

현재 에이전트 상태와 마지막 프롬프트/응답 내용을 파일로 저장합니다.

- 저장 위치: `~/.presence/reports/report-날짜시간.md`
- macOS에서는 파일 저장과 동시에 클립보드에도 복사됩니다.
- 저장 완료 후 경로가 화면에 표시됩니다.

무언가 잘못됐을 때 내용을 보관하거나 지원 요청 시 첨부할 때 사용합니다.

---

### `/quit` / `/exit` — 앱 종료

```
/quit
/exit
```

presence TUI를 종료합니다. 두 커맨드는 동일하게 동작합니다.

---

## 3. `/memory` — 기억 관리

presence는 대화 중 중요한 내용을 자동으로 기억합니다. `/memory` 커맨드로 저장된 기억을 조회하거나 삭제할 수 있습니다.

### 하위 커맨드 목록

| 입력 | 동작 |
|------|------|
| `/memory` | 현재 저장된 기억 개수 요약 표시 |
| `/memory list` | 최근 기억 최대 30개 목록 표시 |
| `/memory clear` | 저장된 기억 전체 삭제 |
| `/memory clear 7d` | 7일보다 오래된 기억만 삭제 |
| `/memory clear 24h` | 24시간보다 오래된 기억만 삭제 |
| `/memory clear 30m` | 30분보다 오래된 기억만 삭제 |
| `/memory help` | `/memory` 도움말 표시 |

### `/memory` (요약)

```
/memory
```

저장된 기억 개수를 한 줄로 표시합니다.

**표시 예시:**

```
메모리: 15개 노드 (15 nodes)
```

### `/memory list` (목록 보기)

```
/memory list
```

저장된 기억을 최근 순으로 최대 30개 표시합니다.

**표시 예시:**

```
  1. 사용자가 Python을 선호한다고 언급  [2h]
  2. 프로젝트 이름은 presence           [1d]
  3. 주요 작업 디렉토리는 ~/projects    [3d]
... +5 more
```

- 대괄호 안 숫자는 기억이 저장된 후 경과한 시간입니다. (`s`=초, `m`=분, `h`=시간, `d`=일)
- 30개를 초과하면 `... +N more` 형태로 나머지 개수를 알려줍니다.

> **주의 — tier 필터는 현재 동작하지 않습니다.** `/memory help`에는 `/memory list episodic` 처럼 종류별 필터 기능이 안내되어 있지만, **현재 구현에서는 종류 인자를 무시하고 항상 전체 목록을 표시합니다.** `/memory list episodic` 을 입력해도 전체 목록이 나옵니다. (UX 감사 FP-3)

### `/memory clear` (기억 삭제)

```
/memory clear          ← 전체 삭제
/memory clear 7d       ← 7일보다 오래된 것만 삭제
/memory clear 24h      ← 24시간보다 오래된 것만 삭제
/memory clear 30m      ← 30분보다 오래된 것만 삭제
```

기간 단위: `d`(일), `h`(시간), `m`(분)

**결과 예시:**

```
5개 노드 삭제
```

> **영구 변경** — 삭제된 기억은 복구할 수 없습니다. 전체 삭제(`/memory clear`) 전에 `/memory list` 로 내용을 먼저 확인하는 것을 권장합니다.

> **참고** — 기간을 지정하여 삭제할 때 완료 메시지 일부가 영어로 표시될 수 있습니다(예: `older than 7d`). 이는 알려진 표시 문제입니다. (UX 감사 FP-4)

## 4. `/sessions` — 세션 관리

**세션**이란 하나의 독립된 대화 공간입니다. 여러 세션을 만들어 주제별로 대화를 분리할 수 있습니다. 각 세션의 대화 이력은 서로 섞이지 않습니다.

### 하위 커맨드 목록

| 입력 | 동작 |
|------|------|
| `/sessions` 또는 `/sessions list` | 세션 목록 표시 |
| `/sessions new` | 이름 없이 새 세션 생성 |
| `/sessions new 이름` | 이름을 지정하여 새 세션 생성 |
| `/sessions switch <id>` | 다른 세션으로 전환 |
| `/sessions delete <id>` | 세션 삭제 |

### `/sessions list` (목록 보기)

```
/sessions
/sessions list
```

**표시 예시:**

```
sessions:
● abc123  [user] (현재)
  def456  [user]
```

- `●` 표시가 있는 것이 현재 접속 중인 세션입니다.
- `[user]`는 일반 대화 세션을 의미합니다.
- 세션 ID는 서버에서 자동 부여한 고유 식별자입니다.

### `/sessions new` (새 세션 만들기)

```
/sessions new
/sessions new work
```

새 세션을 만들고 생성된 세션 ID를 화면에 표시합니다.

`/sessions new work` 처럼 이름을 붙이면, 그 이름이 그대로 **세션 ID**가 됩니다. presence에는 별도의 "표시 이름" 개념이 없으며, `/sessions new` 에서 입력한 값이 ID이자 이름입니다. 이후 `/sessions switch work`, `/sessions delete work` 처럼 그대로 사용합니다.

> **새 세션으로 자동 이동하지 않습니다.** 새 세션을 만든 뒤 그 세션을 사용하려면 `/sessions switch <id>` 로 직접 이동해야 합니다.

### `/sessions switch` (세션 전환)

```
/sessions switch def456
```

다른 세션으로 전환합니다. "세션 전환 중..." 메시지가 표시된 후 화면이 잠시 다시 연결됩니다.

> **주의 — 완료 확인 방법** — 전환이 완료돼도 별도의 완료 메시지가 표시되지 않습니다. (UX 감사 FP-2) `/sessions list` 를 입력하여 `●` 표시가 원하는 세션 앞으로 이동했는지 확인하세요. 재연결에는 수 초가 걸릴 수 있으니 잠시 기다린 후 확인하는 것이 좋습니다.

### `/sessions delete` (세션 삭제)

```
/sessions delete def456
```

세션과 그 안의 대화 이력을 영구 삭제합니다.

- **현재 접속 중인 세션은 삭제할 수 없습니다.** 다른 세션으로 전환한 후 삭제하세요.

> **영구 변경** — 삭제된 세션과 대화 이력은 복구할 수 없습니다.

## 5. `/statusline` — 상태바 항목 설정

화면 아래에는 현재 상태를 보여주는 **상태바**가 있습니다. 어떤 항목을 표시할지 직접 설정할 수 있습니다.

### 사용법

```
/statusline                ← 현재 표시 중인 항목과 추가 가능한 항목 확인
/statusline +항목이름      ← 항목 추가
/statusline -항목이름      ← 항목 제거
```

**현재 상태 확인 예시:**

```
statusline items: budget, model, dir, branch (status: always on)
available: turn, mem, tools
usage: /statusline +item  /statusline -item
```

### 토글 가능한 항목 목록

| 항목 이름 | 표시 내용 |
|-----------|----------|
| `session` | 현재 세션 ID (예: `session: work`) |
| `turn` | 대화 횟수 (예: `turn: 5`) |
| `mem` | 저장된 기억 개수 (예: `mem: 12`) |
| `tools` | 사용 가능한 도구 개수 (예: `tools: 8`) |
| `budget` | 프롬프트 사용률 (예: `budget: 42%`) |
| `dir` | 현재 작업 폴더 이름 |
| `branch` | 현재 Git 브랜치 이름 |
| `model` | 현재 사용 중인 AI 모델 이름 |

> **`status`는 항상 표시됩니다.** 에이전트가 대기 중인지(`● idle`), 처리 중인지(`⠙ thinking...`) 나타내는 항목으로, 제거할 수 없습니다.

**에러가 발생하면** 상태바의 `status` 항목이 에러 분류와 함께 표시됩니다.

```
✗ error: interpreter
```

| 에러 분류 | 뜻 |
|-----------|---|
| `planner_parse` | AI 응답 형식을 읽지 못했습니다 |
| `planner_shape` | AI 응답 구조가 예상과 다릅니다 |
| `interpreter` | 도구 실행 중 문제가 발생했습니다 |
| `max_iterations` | 처리 횟수 상한에 도달했습니다 |

분류만으로는 원인을 파악하기 어려울 때는 `/report` 로 상세 내용을 파일로 저장하여 확인하세요.

### 사용 예시

```
/statusline +turn          ← 대화 횟수 항목 추가
/statusline -branch        ← Git 브랜치 항목 제거
/statusline                ← 변경 후 전체 구성 확인
```

> **기본 표시 항목** — 앱을 시작하면 `session`, `budget`, `model`, `dir`, `branch` 다섯 항목이 기본으로 표시됩니다. 앱을 재시작하면 이 기본 구성으로 돌아갑니다.

## 6. `/mcp` — MCP 서버 관리

**MCP(Model Context Protocol)**는 에이전트가 외부 도구를 사용할 수 있게 해주는 연결 방식입니다. `/mcp` 커맨드로 연결된 MCP 서버 목록을 확인하고, 각 서버를 활성화하거나 비활성화할 수 있습니다.

### 사용법

```
/mcp                       ← 연결된 MCP 서버 목록 조회 (기본값)
/mcp list                  ← 위와 동일
/mcp enable mcp0           ← mcp0 서버 활성화
/mcp disable mcp0          ← mcp0 서버 비활성화
```

### 목록 표시 예시

```
MCP servers:
● mcp0  github-tools  (5 tools)
○ mcp1  browser-tools  (3 tools)
```

- `●` 는 현재 활성화된 서버, `○` 는 비활성화된 서버입니다.
- 괄호 안 숫자는 해당 서버가 제공하는 도구 개수입니다.
- 서버 ID(예: `mcp0`, `mcp1`)는 `enable`/`disable` 시 사용합니다.

> **설정된 MCP가 없을 때** — `No MCP servers configured.` 메시지가 표시됩니다. MCP 서버 추가는 설정 파일에서 합니다.

## 7. 입력 히스토리 — 이전 커맨드 다시 불러오기

입력창에서 키보드 **위/아래 방향키(↑ / ↓)** 를 누르면 이전에 입력했던 내용을 다시 불러올 수 있습니다.

- `↑` — 더 이전에 입력한 내용으로 이동
- `↓` — 더 최근에 입력한 내용으로 이동

자주 쓰는 커맨드(예: `/memory list`, `/sessions list`)를 반복해서 사용할 때 편리합니다.

---

## 8. 알려진 제한

### 오타 커맨드가 에이전트로 전달됨

`/`로 시작하더라도 등록되지 않은 커맨드는 **에이전트에게 그대로 전달되어 대화가 시작됩니다.** 오류 메시지가 표시되지 않으므로 주의가 필요합니다.

| 잘못된 입력 | 실제 동작 | 올바른 입력 |
|-------------|----------|------------|
| `/mem` | 에이전트에 전달됨 | `/memory` |
| `/model` | 에이전트에 전달됨 | `/models` |
| `/session` | 에이전트에 전달됨 | `/sessions` |

에이전트가 예상치 못한 반응을 보이면 입력한 커맨드의 철자를 다시 확인하세요. 전체 커맨드 목록은 이 문서 상단의 표나 `/help` 로 확인할 수 있습니다. (UX 감사 FP-7)
